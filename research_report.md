# CoAI (ChatNio) 深度架构与原理研究报告

## 1. 项目概述
**CoAI** 是一个基于 Golang 和 React 构建的下一代 AIGC 一站式商业解决方案。它集成了对话 UI、多模型分发中转、计费管理、订阅体系和强大的渠道管理功能，旨在打通 B 端 API 分发与 C 端对话应用的壁垒。

- **核心定位**：Next Web（美观 UI）+ One API（强大中转）的结合体。
- **技术栈**：
    - **前端**：React, Redux, Tailwind CSS, Radix UI, Vite, PWA, Tauri (桌面端)。
    - **后端**：Golang (Gin 框架), Redis (缓存/速率限制), MySQL (持久化)。
    - **通信**：HTTP (RESTful API), WebSocket (对话流通信)。

---

## 2. 逻辑架构分析

### 2.1 模块化设计
项目后端采用了清晰的模块化结构，各司其职：
- **`adapter/`**：核心适配层。将各种模型服务商（OpenAI, Claude, Gemini, Midjourney 等）的私有接口统一抽象为标准的内部调用格式。
- **`manager/`**：业务逻辑中心。处理对话流（`chat.go`）、HTTP 中转请求（`relay.go`）、图像生成（`images.go`）等。
- **`channel/`**：渠道管理层。负责上游渠道的优先级排序、权重分配、健康检测及失败重试机制。
- **`auth/`**：账户与计费系统。管理用户配额（Quota）、订阅计划（Subscription）及支付鉴权。
- **`addition/`**：附加功能层。如全模型联网搜索（基于 SearXNG）、文件解析等。

### 2.2 数据流向
1. **客户端发起请求**：通过 WebSocket (`/chat`) 或 HTTP (`/v1/chat/completions`) 进入。
2. **中间件校验**：`middleware` 进行身份验证、IP 速率限制及用户余量预检。
3. **路由分发**：请求被派发到 `manager` 层的对应处理器。
4. **渠道选择**：`channel.Manager` 根据模型名称和用户组，从可用渠道池中选出最优上游。
5. **协议适配**：`adapter` 将请求转换为目标 API 的格式并发送。
6. **响应回传**：通过流式（Streaming）或同步方式返回给用户，并实时计算 Token 消耗扣除费用。

## 3. 核心原理深入分析

### 3.1 负载均衡与调度算法 (`Ticker` 机制)
CoAI 采用了一种基于 **优先级 + 权重** 的两级调度算法：
- **优先级 (Priority)**：渠道按优先级从高到低排序。系统首选最高优先级的渠道。
- **权重 (Weight)**：在同一优先级内，使用 **加权随机算法** 选择渠道。
- **容错重试**：如果选中的渠道请求失败，`Ticker` 会自动跳到下一优先级的渠道进行重试，确保高可用性。

### 3.2 计费与订阅模型
项目支持两种并行的商业模式：
- **弹性计费 (Quota)**：
    - 数据库存储 `quota`（可用点数）和 `used`（已用点数）。
    - 比例固定为 `10 点数 = 1 元`（内部汇率可自定义）。
    - 支持按 Token 或按次计费。
- **订阅制 (Subscription)**：
    - 划分为四个等级：普通、基础、标准、专业。
    - 订阅等级直接映射为用户组，可控制不同模型的使用权限。
    - 支持自动降级与升级时的差价计算。

### 3.3 对话同步与缓存
- **缓存机制**：使用 Redis 对请求参数进行 Hash。对于相同的请求，如果开启了模型缓存，将直接返回之前的结果（Hit），从而降低成本并提高响应速度。
- **跨端同步**：对话记录存储在 MySQL 中。通过用户体系，用户可以在 Web、移动端、桌面端（Tauri）无缝同步聊天记录，无需依赖 WebDav。

### 3.4 联网搜索原理
- **SearXNG 集成**：当用户开启联网搜索时，系统先调用 SearXNG 聚合搜索引擎获取网页内容。
- **Prompt 增强**：将搜索到的结果（标题、链接、摘要）格式化后注入到 AI 的 System Prompt 中，从而使模型具备处理实时信息的能力。

## 4. 部署与运维

- **容器化**：提供完善的 `Dockerfile` 和 `docker-compose` 配置，支持 MySQL、Redis 与后端一键拉起。
- **静态资源**：后端内置静态文件服务开关（`SERVE_STATIC`），支持前后端合并部署或分离部署（如前端部署在 Vercel/Nginx，后端在 Docker）。
- **扩展性**：支持 Zeabur、阿里云计算巢等平台的一键托管，具备良好的 SEO 优化能力。

---

## 5. 总结与评价
**CoAI** 是一款工程化程度极高的 AI 中转管理系统。它的优势在于：
1. **极致的 UI 体验**：弥补了 One API 类项目界面简陋的短板。
2. **闭环的商业化能力**：内置了成熟的支付对账、礼品码、兑换码、订阅分层系统。
3. **强大的技术适配**：通过 `adapter` 设计，几乎兼容了市面上所有的主流大模型。

对于想要快速搭建 AI 商业平台的开发者来说，这是一个兼顾广度（模型支持）与深度（计费调度）的优秀参考架构。
